# Debug Journal

## Purpose

This document records confirmed, reusable engineering lessons that prevent repeated mistakes. It is not a source of current runtime authorization, not a replacement for PHASE_STATUS, and not a place for raw server logs.

## Entry rules

- Record only confirmed incidents or confirmed engineering lessons.
- Do not include secrets, tokens, credentials, private IPs, public IPs, customer identifiers, raw firewall output, raw database output, complete evidence bundles, or large command logs.
- Link to a regression test, code change, PR, or canonical document where possible.
- Do not infer current server state from an old incident.
